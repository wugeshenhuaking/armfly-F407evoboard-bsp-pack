# 源码文件编码处理规则

本工程为安富莱 STM32F407 例程，源码存在混合编码（UTF-8 与 GB2312/GBK 并存）。
编辑/生成代码时必须遵循以下约定，以避免中文乱码：

1. **写入编码**：所有新建 / 重写 / 替换的文本一律使用 **UTF-8（无 BOM）**。
2. **遇到 GB2312/GBK 文件**：修改前，先用原编码（GB2312/GBK）正确解码，再以 UTF-8（无 BOM）重新写回；
   严禁把 UTF-8 字节直接塞进 GB2312 文件（会导致中文乱码）。
3. **跳过第三方文件（不参与扫描 / 修改 / 编码转换，保留原样）**：
   以下目录为第三方代码，除非用户明确要求，否则不扫描、不修改、不转码：
   - `Libraries/` —— ST HAL 库
   - `User/segger/` —— SEGGER 中间件
   - （今后新增第三方库，请在此清单继续追加目录名，例如 `Middlewares/`、`User/lwip/`、`User/fatfs/` 等）
4. **只处理用户自写文件**：编码转换 / 统一仅针对 `User/` 目录下安富莱 / 用户自写的 `.c/.h` 文件。
5. **判断编码的方法**：用严格 UTF-8 解码（`UTF8Encoding(false, true)`）测试，不抛异常即为 UTF-8；
   否则尝试 GB2312 解码，确认为 GBK 系后再做转换。
6. **中文字符串常量**（如 LCD 显示用）：新增时编码需与对应字库一致，必要时提醒用户。

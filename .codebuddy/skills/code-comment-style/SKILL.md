# 代码注释风格规范化（安富莱 V5 风格）

## 用途
将任意 C 源文件调整为安富莱 V5 BSP 包统一的注释与排版规范，便于团队阅读和后续维护。

## 何时使用
- 用户要求"统一注释风格""规范化注释""整理代码格式"。
- 用户要求补齐 / 修正文件头注释、函数注释块。
- 用户要求确认或修改文件编码、缩进。

## 规范细则
1. **文件头星框**：每个 `.c` / `.h` 顶部使用如下结构（`*` 对齐的边框）：
   ```
   /*
   *********************************************************************************************************
   *
   *   模块名称 : XXX 驱动模块
   *   文件名称 : bsp_xxx.c
   *   版    本 : V1.0
   *   说    明 : 简要说明
   *
   *   修改记录 :
   *       版本号  日期        作者     说明
   *       V1.0    2026-xx-xx 用户     正式发布
   *
   *   Copyright (C), 2026, 安富莱电子 www.armfly.com
   *
   *********************************************************************************************************
   */
   ```
2. **函数注释块**：每个函数前用星框注释，固定字段为：`函 数 名`、`功能说明`、`形    参`（或 `形    参：`）、`返 回 值`。
3. **缩进**：使用 **Tab，宽度 4**；不要混用空格与 Tab。
4. **注释符号**：使用 `/* */`，不推荐使用 `//`（现有 BSP 文件均用 `/* */`）。
5. **文件末尾**：追加一行 `/***************************** 安富莱电子 www.armfly.com (END OF FILE) *********************************/`。
6. **编码**：所有文本使用 **UTF-8（无 BOM）**；若原文件为 GB2312/GBK，按 `rules/encoding.md` 先转 UTF-8 再编辑，严禁直接写入 UTF-8 字节导致中文乱码。
7. **第三方文件跳过**：`Libraries/`（ST HAL 库）、`User/segger/` 等第三方目录不改动，除非用户明确要求。

## 使用步骤
1. 读取目标文件，识别当前风格偏差。
2. 按上述细则重写 / 补全文件头、函数注释、缩进与末尾行。
3. 校验编码为 UTF-8 无 BOM；若不是，先转码再保存。
4. 保持代码逻辑与英文标识符不变，仅调整注释与排版。
